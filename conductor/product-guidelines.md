# Product Guidelines

These guidelines define the prose style, branding tone, and user experience principles for Password Exchange to ensure a consistent and trustworthy user experience.

## Prose Style: Authoritative and Confident
- Use a direct, professional, and confident tone throughout the documentation and user-facing text.
- Avoid overly conversational filler; state the facts and instructions clearly.
- Use active voice (e.g., "The system encrypts the data" instead of "The data is encrypted by the system").

## Branding Tone: Security-First and Trustworthy
- Prioritize clear communication about security features and privacy protections.
- Use terminology that reinforces trust (e.g., "secure storage," "end-to-end encryption," "automatic expiration").
- Branding should feel utilitarian, professional, and highly reliable.

## User Experience Principles
### Simplicity and Speed
- Minimize the number of steps required for core actions like submitting or retrieving a secret.
- Interfaces should be clean, focused, and free of unnecessary distractions.

### Platform Consistency
- Ensure a consistent experience across all interfaces: Web UI, REST API, and Slackbot.
- Use consistent naming conventions and terminology for all features (e.g., "Secret," "Decryption Key," "Expiration").

## Technical Writing and API Documentation
### Action-Oriented Examples
- Prioritize practical, copy-pasteable examples for common API use cases.
- Use clear, concise explanations for parameters and error codes.
- Focus on helping developers integrate with the platform as quickly as possible.
